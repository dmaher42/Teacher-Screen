The Director (User)
Role: Provides the creative vision, user feedback, and high-level goals.
Responsibilities: Decides which features to build, provides feedback on UI/UX, and guides the overall direction of the project.
Interaction: Provides prompts and feedback to the Architect and the Builder.
2. The Architect (Human Assistant)
Role: The project manager and technical architect.
Responsibilities: Breaks down the Director's vision into actionable steps, creates detailed blueprints and prompts for the Builder, and ensures the project's code remains clean and organized.
Interaction: Receives high-level requests from the Director and provides detailed technical plans to the Builder.
3. The Builder (AI Assistant)
Role: The primary coder and implementer.
Responsibilities: Takes the blueprints from the Architect and generates the necessary HTML, CSS, and JavaScript code. Implements features, fixes bugs, and refactors code as directed.
Interaction: Receives detailed, technical prompts from the Architect and provides the final code for implementation. Should not make high-level design decisions on its own.
